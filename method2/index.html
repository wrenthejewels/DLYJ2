V2.0 Methodology

Current Model Math

This page documents the live 2.0 role-transformation model as currently implemented in v2_engine.js. It excludes the archived legacy timing model entirely.

Model Scope

The live main model is occupation-anchored and transformation-first. It answers five questions: what work is exposed, whether exposed work looks more like augmentation or automation, whether the retained bundle still forms a coherent role, how well the user's profile fits the retained role, and what labor-market context surrounds that occupation.

The public result now also exposes a task-level breakdown. That task view is not a separate model: it is a projection of the selected occupation's O*NET task list through the same occupation anchor, task-cluster weights, questionnaire signals, and Anthropic evidence stack used by the headline outputs.

The public Role Transformation Map is now derived from those live task rows rather than from the raw shared cluster bundle alone. The map ranks the mapped tasks by current share, exposed share, and retained share so the displayed rows are occupation-specific.

The current scoring stack uses O*NET occupation structure, Anthropic Economic Index task priors, derived occupation priors, and BLS labor-market context. The live headline outputs do not use the archived hazard/timing math.

Runtime Inputs

The engine resolves the user to a concrete occupation and then applies direct task and questionnaire inputs.

Input tuple
$$x = (\text{occupationId},\ \text{roleCategory},\ \text{answers}_{1:19},\ \text{seniorityLevel},\ \text{dominantTaskClusters},\ \text{roleCriticalClusters},\ \text{aiToolSupportLevel},\ \text{residualRoleDistinctiveness})$$

The engine then loads normalized occupation task clusters, occupation tasks, task-cluster membership rows, task-level transformation evidence, cluster priors, occupation exposure priors, adaptation priors, labor-market context, and mapped unemployment time series.

Task-Bundle Reconstruction

Let the baseline occupation share for cluster i be $s_i^{(0)}$. The current implementation now treats the occupation bundle as a prior composition with concentration $\kappa$, then updates that prior with pseudo-mass from direct task inputs.

Pseudo-count bundle update
$$o_i = w_i^{\text{explicit}} + 0.16 \cdot \mathbf{1}[i \in \text{dominantTaskClusters}] + 0.10 \cdot \mathbf{1}[i \in \text{roleCriticalClusters}]$$
$$\kappa = 0.75 + 1.5 \cdot \operatorname{avg}_i(\text{clusterEvidenceConfidence}_i)$$
$$s_i = \frac{\kappa s_i^{(0)} + o_i}{\kappa + \sum_j o_j}$$

This is equivalent to a Dirichlet-style prior update rather than an unconstrained ad hoc rescaling. In the current release, the direct inputs still tilt the occupation prior rather than fully replacing it with user-supplied time shares.

Occupation Assignment And Task Projection

The engine always scores a concrete mapped occupation. The selected category narrows the candidate set, occupation search can resolve directly to a mapped launch occupation, and the final role briefing is anchored to that occupation's O*NET/SOC identity.

Occupation anchor strength
$$C_{\text{anchor}} = \operatorname{avg}(\text{occupationPriorConfidence},\ \text{selectorWeight})$$

In the interface this is labeled Occupation anchor strength. It is a descriptive anchor score, not a semantic search confidence or a posterior probability over occupation identity.

Once the occupation is fixed, the task panel is generated from its O*NET task list. Let task t belong to cluster $c(t)$. The current implementation constructs a within-cluster task weight from O*NET importance, O*NET frequency, observed Anthropic usage when available, and a small core-task boost:

Task weighting inside a cluster
$$w_t^{\text{raw}} = \max\!\left(0.04,\ 0.58I_t + 0.27F_t + 0.15U_t + 0.08\mathbf{1}[t \in \text{core tasks}]\right)\cdot m_t$$
$$s_t = s_{c(t)} \cdot \frac{w_t^{\text{raw}}}{\sum_{j \in c(t)} w_j^{\text{raw}}}$$

Here $I_t$ is normalized task importance, $F_t$ is normalized task frequency, $U_t$ is observed Anthropic usage share when present, and $m_t$ is the task-cluster membership confidence weight. This means questionnaire-driven cluster weights flow through to the task view automatically.

Questionnaire Signals

The visible questionnaire is grouped for 2.0, but internally the engine still maps the answers into normalized $q_1$ through $q_{19}$ values.

Normalization
$$q_k = \operatorname{clamp}\!\left(\frac{Q_k - 1}{4},\,0,\,1\right),\qquad s = \operatorname{clamp}\!\left(\frac{\text{seniorityLevel}-1}{4},\,0,\,1\right),\qquad r \in [0,1]$$
Composite signals
$$E = \operatorname{avg}(q_1,q_2,q_3,q_4,q_5,q_6,q_8)$$
$$C = \operatorname{avg}(q_7,q_9,q_{11},q_{12})$$
$$A_{\text{auto}} = \operatorname{avg}(q_3,q_4,q_5,q_6,q_8,1-q_7,1-q_9,1-q_{11},1-q_{12})$$
$$A_{\text{aug}} = \operatorname{avg}(q_1,q_4,q_8,q_7,q_9,q_{11})$$
$$P_{\text{adopt}} = \operatorname{avg}(q_{13},q_{14},1-q_{15},q_{16})$$
$$F_{\text{adapt}} = \operatorname{avg}(q_{17},q_{18},q_{19})$$
$$G_{\text{fragile}} = \operatorname{avg}(q_{10},1-r)$$

P_adopt and the direct AI support signal are now part of the live scoring path, but they still act as secondary modifiers rather than replacing task evidence.

Cluster-Level Transformation Scoring

For each task cluster i, the engine loads a base exposure prior $e_i^{(0)}$, a base augmentation prior $a_i^{(0)}$, and a base automation prior $u_i^{(0)}$. The automation prior is the mean of the partial-automation and high-automation priors. These cluster priors are no longer averaged mechanically with occupation priors; they are shrunk toward occupation-level priors using an adjusted reliability term.

Let $h_i$ be the built-in human-advantage weight for cluster i, let $k_i = \mathbf{1}[i \in \text{roleCriticalClusters}]$, and let $T$ be the direct AI/tool support signal. The live formulas are:

Prior shrinkage and transformation
$$\rho_i = \text{adjusted cluster-prior reliability} \in [0,1]$$
$$\bar{e}_i^{(0)} = \rho_i e_i^{(0)} + (1-\rho_i)E_{\text{occ}}$$
$$\bar{a}_i^{(0)} = \rho_i a_i^{(0)} + (1-\rho_i)A_{\text{occ}}$$
$$\bar{u}_i^{(0)} = \rho_i u_i^{(0)} + (1-\rho_i)U_{\text{occ}}$$
$$R_{\text{adopt}} = 0.92 + 0.16P_{\text{adopt}}$$
$$e_i = \operatorname{clamp}\!\left(\bar{e}_i^{(0)} \cdot (0.68 + 0.62E)\cdot R_{\text{adopt}}\cdot(1 + 0.08k_i)\cdot(1 - 0.42 h_i C),\,0,\,1\right)$$
$$a_i = \operatorname{clamp}\!\left(\bar{a}_i^{(0)} \cdot (0.72 + 0.48A_{\text{aug}} + 0.32T)\cdot(1 + h_i(0.16 + 0.12T))\cdot R_{\text{adopt}},\,0,\,1\right)$$
$$u_i = \operatorname{clamp}\!\left(\bar{u}_i^{(0)} \cdot (0.68 + 0.68A_{\text{auto}} + 0.18P_{\text{adopt}})\cdot(1 - h_i(0.42C + 0.18T)),\,0,\,1\right)$$
$$m_i = \arg\max\{a_i,\ u_i\}$$

The model then estimates absorbed, retained, and elevated share for each cluster.

Bundle transformation
$$\text{absorbed}_i = s_i \cdot e_i \cdot u_i \cdot (0.74 + 0.16P_{\text{adopt}})$$
$$\text{retained}_i = \max(0,\ s_i - \text{absorbed}_i)$$
$$\text{elev}_i = \begin{cases} \text{absorbed}_i \cdot (0.22 + 0.18s + 0.12r + 0.08k_i), & i \in \mathcal{E} \\ 0, & i \notin \mathcal{E} \end{cases}$$
$$t_i = \text{retained}_i + \text{elev}_i$$

Here $\mathcal{E}$ is the fixed set of elevation-capable clusters: coordination, QA/review, decision support, oversight/strategy, relationship management, and client interaction. A user-selected role-defining cluster receives extra weight in exposure and elevation so the model is not driven by time share alone.

Task-Level Exposure Projection

The task panel reuses the cluster-level transformation math rather than introducing a second scoring system. For each task t, the engine first takes the cluster-level result for $c(t)$ and then shrinks direct Anthropic task evidence toward that cluster estimate using task-count-weighted reliability.

Task-level exposure and mode
$$\lambda_t = c_t \cdot \frac{n_t}{n_t + 40}$$
$$e_t = \begin{cases} \lambda_t e_t^{\text{direct}} + (1-\lambda_t)e_{c(t)}, & \text{if direct task evidence exists} \\ e_{c(t)}, & \text{otherwise} \end{cases}$$
$$a_t = \begin{cases} \lambda_t a_t^{\text{direct}} + (1-\lambda_t)a_{c(t)}, & \text{if direct task evidence exists} \\ a_{c(t)}, & \text{otherwise} \end{cases}$$
$$u_t = \begin{cases} \lambda_t u_t^{\text{direct}} + (1-\lambda_t)u_{c(t)}, & \text{if direct task evidence exists} \\ u_{c(t)}, & \text{otherwise} \end{cases}$$
Task transformation
$$\text{absorbed}_t = s_t \cdot e_t \cdot u_t \cdot (0.74 + 0.16P_{\text{adopt}})$$
$$\text{retained}_t = \max(0,\ s_t - \text{absorbed}_t)$$
$$\text{elev}_t = \begin{cases} \text{absorbed}_t \cdot (0.22 + 0.18s + 0.12r + 0.08k_{c(t)}), & c(t) \in \mathcal{E} \\ 0, & c(t) \notin \mathcal{E} \end{cases}$$
$$t_t = \text{retained}_t + \text{elev}_t$$

Here $c_t$ is task-level evidence confidence and $n_t$ is the parsed Anthropic task count when available. In the interface, the task list is sorted by exposed share $s_t e_t$ by default. The user can expand from the top ten rows to the full mapped task list without changing the underlying calculations.

The public map is then derived from this same task table:

Task-derived public map
$$\text{Current map rows} = \operatorname{TopK}_t(s_t)$$
$$\text{Exposed map rows} = \operatorname{TopK}_t(s_t e_t)$$
$$\text{Residual map rows} = \operatorname{TopK}_t(t_t)$$

This means the visible map now shows the occupation's own mapped tasks rather than a fixed generic family list.

Recomposition Layer

The live result also includes a secondary recomposition read. This is a descriptive structural layer, not a separate labor-demand forecast.

Explicit task-family frictions
$$F_i = \{x_i,\ a_i,\ j_i,\ d_i,\ t_i\}$$
$$x_i=\text{exception burden},\quad a_i=\text{accountability load},\quad j_i=\text{judgment requirement},\quad d_i=\text{document intensity},\quad t_i=\text{tacit/context dependence}$$

These friction dimensions are derived from the questionnaire and blended with reviewed task-family profiles before the engine scores exposure, augmentation, automation, and recomposition.

Workflow compression
$$W_{\text{raw}} = \sum_i s_i e_i \cdot \operatorname{clamp}\!\left(u_i + (1-u_i)g_i \cdot 0.42,\ 0,\ 1\right)$$
$$P_{\text{dep}} = 1.10 \sum_{(i,j)\in\mathcal{E}} d_{ij}\cdot \min(s_i,s_j)\cdot (1-\kappa_i)(1-\kappa_j)$$
$$W = \operatorname{clamp}\!\left(\max(0, W_{\text{raw}}-P_{\text{dep}}) \cdot (1 - 0.20C),\,0,\,1\right)$$
Organizational conversion and gap
$$O = \operatorname{clamp}\!\left(0.20P_{\text{adopt}} + 0.14\pi_{\text{auto}} + 0.12G_{\text{fragile}} + 0.12(1-C) + 0.08(1-r) + 0.05(1-T) + 0.12S_{\text{exposed}} + 0.06\bar d + 0.05(1-\bar a) + 0.03(1-\bar j) + 0.03(1-\bar t),\,0,\,1\right)$$
$$S_{\text{sub}} = W \cdot O,\qquad G_{\text{recomp}} = W - S_{\text{sub}}$$
Uncertainty bands
$$c_{\text{recomp}} = \operatorname{avg}(C_{\text{task}}, C_{\text{anchor}}, C_{\text{personal}}, R_{\text{direct}}, 1-0.45\cdot P_{\text{dep}}/W_{\text{raw}})$$
$$h = \operatorname{clamp}(0.06 + 0.18(1-c_{\text{recomp}}) + 0.05(1-R_{\text{direct}}) + 0.04(1-C_{\text{anchor}}),\,0.06,\,0.24)$$
$$[z^{-}, z^{+}] = [\operatorname{clamp}(z-h,0,1),\ \operatorname{clamp}(z+h,0,1)]$$

In the interface these are shown as Workflow compression, Organizational conversion, Substitution potential, and Recomposition gap. They are displayed as banded descriptive values with explicit uncertainty ranges, not as precise estimates of employment loss.

Role-Level Aggregation

Cluster mass is aggregated into exposed share, automation/augmentation balance, strategic retained share, residual viability, and personalization fit.

Exposure and balance
$$S_{\text{exposed}} = \operatorname{clamp}\!\left(\sum_i s_i e_i,\ 0,\ 1\right)$$
$$M_{\text{auto}} = \sum_i s_i e_i u_i,\qquad M_{\text{aug}} = \sum_i s_i e_i a_i$$
$$\pi_{\text{auto}} = \begin{cases} \frac{M_{\text{auto}}}{M_{\text{auto}} + M_{\text{aug}}}, & M_{\text{auto}} + M_{\text{aug}} > 0 \\ 0.5, & \text{otherwise} \end{cases},\qquad \pi_{\text{aug}} = 1-\pi_{\text{auto}}$$
Strategic retained share
$$S_{\text{strategic}} = \sum_{i \in \mathcal{E} \cap \mathcal{R}} t_i$$
$$S_{\text{critical,exposed}} = \sum_{i \in \mathcal{K}} s_i e_i,\qquad S_{\text{critical,retained}} = \sum_{i \in \mathcal{K}} t_i$$
$$B_{\text{critical}} = \begin{cases} \operatorname{avg}\!\left(S_{\text{critical,retained}},\ 1 - \operatorname{clamp}(S_{\text{critical,exposed}}\cdot\max(\pi_{\text{auto}},0.45),0,1)\right), & |\mathcal{K}| > 0 \\ r, & |\mathcal{K}| = 0 \end{cases}$$
Residual role strength
$$V = \operatorname{clamp}\!\left(0.68 \cdot \operatorname{avg}(1-G_{\text{fragile}},\ r,\ C,\ S_{\text{strategic}},\ B_{\text{critical}},\ 1-\pi_{\text{auto}}) + 0.32 \cdot \operatorname{clamp}\!\left(\sum_i t_i,\ 0,\ 1\right),\ 0,\ 1\right)$$
Personalization fit
$$P = \operatorname{clamp}\!\left(\operatorname{avg}(A_{\text{occ}},\ S_{\text{strategic}},\ S_{\text{critical,retained}} \text{ or } r,\ C,\ F_{\text{adapt}},\ T,\ 0.55s + 0.45r,\ 1-G_{\text{fragile}}),\ 0,\ 1\right)$$

$A_{\text{occ}}$ is the occupation-level adaptive-capacity prior from the adaptation-prior layer when available, otherwise from the occupation-prior layer.

Discrete Output Rules

The live engine uses rule-based thresholds for the five main outputs.

Role outlook

Decision tree
$$\text{Role Outlook} = \begin{cases} \text{High displacement risk}, & V < 0.34 \land \pi_{\text{auto}} \ge 0.58 \land S_{\text{exposed}} \ge 0.58 \\ \text{Role fragments}, & V < 0.42 \land S_{\text{exposed}} \ge 0.55 \\ \text{Role becomes more senior}, & S_{\text{strategic}} \ge 0.34 \land \pi_{\text{auto}} < 0.48 \land s \ge 0.50 \\ \text{Mostly augmented}, & V \ge 0.62 \land \pi_{\text{aug}} \ge 0.55 \\ \text{Routine work compressed}, & V \ge 0.48 \\ \text{Role becomes narrower}, & \text{otherwise} \end{cases}$$

Top exposed work

Selection rule
$$i^\star = \arg\max_i\left((s_i e_i)\cdot(1 + 0.35k_i)\right)$$
$$\text{Exposure tier}(i^\star) = \begin{cases} \text{high}, & e_{i^\star} \ge 0.68 \\ \text{moderate}, & e_{i^\star} \ge 0.40 \\ \text{low}, & \text{otherwise} \end{cases}$$

Mode of change

Balance thresholds
$$\text{Mode} = \begin{cases} \text{mostly automation}, & \pi_{\text{auto}} \ge 0.60 \\ \text{mostly augmentation}, & \pi_{\text{auto}} \le 0.42 \\ \text{mixed}, & \text{otherwise} \end{cases}$$

Residual role strength and personalization fit

Tiering rule
$$\text{Tier}(z) = \begin{cases} \text{strong}, & z \ge 0.68 \\ \text{moderate}, & z \ge 0.45 \\ \text{weak}, & \text{otherwise} \end{cases}$$

Confidence and Evidence Summary

The model emits multiple descriptive evidence signals instead of one monolithic uncertainty score.

Evidence and signal fields
$$C_{\text{task}} = \operatorname{avg}_i(\text{clusterEvidenceConfidence}_i)$$
$$C_{\text{anchor}} = \operatorname{avg}(\text{occupationPriorConfidence},\ \text{selectorWeight})$$
$$C_{\text{personal}} = \operatorname{avg}(r,\ 1-G_{\text{fragile}},\ C,\ T,\ C_{\text{task}})$$
$$C_{\text{labor}} = \text{laborMarketConfidence}$$

In the interface these appear as:

  • Evidence strength = $C_{\text{task}}$
  • Occupation anchor strength = $C_{\text{anchor}}$
  • Personalization signal strength = $C_{\text{personal}}$

The evidence payload also tracks task-row coverage directly:

Task coverage counts
$$N_{\text{direct}} = \#\{t : \text{direct Anthropic task evidence exists}\},\qquad N_{\text{fallback}} = \#\{t : \text{cluster fallback is used}\}$$
$$R_{\text{direct}} = \frac{N_{\text{direct}}}{N_{\text{direct}} + N_{\text{fallback}}}$$
$$\bar{\rho} = \operatorname{avg}_i(\rho_i),\qquad \bar{\lambda} = \operatorname{avg}_{t \in \text{direct rows}}(\lambda_t)$$

Task coverage in the UI is the direct coverage ratio $R_{\text{direct}}$ plus the raw direct/fallback counts. These values summarize coverage and alignment; they are not posterior probabilities over the labels.

The recomposition panel now adds a separate structural confidence read. That confidence is driven by direct task coverage, occupation-anchor strength, personalization signal strength, and the size of the remaining dependency bottlenecks.

Labor-Market Context

BLS data is loaded alongside the transformation result but is not a direct driver of the main role labels. The current engine treats labor context as a separate panel.

Current design rule
$$\text{Main role labels} \perp \text{labor-market panel}$$

The context payload includes employment, annual openings, median wage, projected growth, latest mapped unemployment rate, and the last 12 monthly unemployment observations for the mapped BLS occupation group.

Current Limitations

  • Direct task inputs are still coarse. They matter more than before, but they still tilt the occupation bundle rather than fully replacing it with user-supplied time shares.
  • Task-level rows are partly projected from cluster math. When direct Anthropic task evidence is missing, the task panel inherits cluster-level exposure and mode estimates rather than estimating each task independently from a fully separate model.
  • Adoption context is still a modifier, not a separate labor-demand model. It changes realized pressure inside the transformation engine, but it does not become an independent forecast system.
  • Coefficients remain hand-tuned. The shrinkage and transformation forms are now more principled, but the constants are still calibrated manually rather than estimated from a labeled validation set.
  • Occupation quality remains critical. The largest benchmark-disagreement roles still need better occupation-prior and task-bundle cleanup.
  • Confidence is descriptive. The model exposes coverage and alignment confidence, not a formal uncertainty distribution over every output.
