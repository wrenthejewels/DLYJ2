V2.0 Methodology

Current Model Math

This page documents the live 2.0 role-transformation model as currently implemented in v2_engine.js. It excludes the archived legacy timing model entirely.

Model Scope

The live main model is occupation-anchored and transformation-first. It answers five questions: what work is exposed, whether exposed work looks more like augmentation or automation, whether the retained bundle still forms a coherent role, how well the user's profile fits the retained role, and what labor-market context surrounds that occupation.

The current scoring stack uses O*NET occupation structure, Anthropic Economic Index task priors, derived occupation priors, and BLS labor-market context. The live headline outputs do not use the archived hazard/timing math.

Runtime Inputs

The engine resolves the user to a concrete occupation and then applies direct task and questionnaire inputs.

Input tuple
$$x = (\text{occupationId},\ \text{roleCategory},\ \text{answers}_{1:19},\ \text{seniorityLevel},\ \text{dominantTaskClusters},\ \text{roleCriticalClusters},\ \text{aiToolSupportLevel},\ \text{residualRoleDistinctiveness})$$

The engine then loads normalized occupation task clusters, task-level transformation priors, occupation exposure priors, adaptation priors, labor-market context, and mapped unemployment time series.

Task-Bundle Reconstruction

Let the baseline occupation share for cluster i be $s_i^{(0)}$. The current implementation applies additive override mass from direct task inputs and then renormalizes.

Task override rule
$$o_i = w_i^{\text{explicit}} + 0.16 \cdot \mathbf{1}[i \in \text{dominantTaskClusters}] + 0.10 \cdot \mathbf{1}[i \in \text{roleCriticalClusters}]$$
$$\tilde{s}_i = s_i^{(0)} + o_i,\qquad s_i = \frac{\tilde{s}_i}{\sum_j \tilde{s}_j}$$

This produces the personalized current bundle shown in the transformation map. In the current release, these are still lightweight tilts on the occupation prior rather than a fully user-specified role reconstruction.

Questionnaire Signals

The visible questionnaire is grouped for 2.0, but internally the engine still maps the answers into normalized $q_1$ through $q_{19}$ values.

Normalization
$$q_k = \operatorname{clamp}\!\left(\frac{Q_k - 1}{4},\,0,\,1\right),\qquad s = \operatorname{clamp}\!\left(\frac{\text{seniorityLevel}-1}{4},\,0,\,1\right),\qquad r \in [0,1]$$
Composite signals
$$E = \operatorname{avg}(q_1,q_2,q_3,q_4,q_5,q_6,q_8)$$
$$C = \operatorname{avg}(q_7,q_9,q_{11},q_{12})$$
$$A_{\text{auto}} = \operatorname{avg}(q_3,q_4,q_5,q_6,q_8,1-q_7,1-q_9,1-q_{11},1-q_{12})$$
$$A_{\text{aug}} = \operatorname{avg}(q_1,q_4,q_8,q_7,q_9,q_{11})$$
$$P_{\text{adopt}} = \operatorname{avg}(q_{13},q_{14},1-q_{15},q_{16})$$
$$F_{\text{adapt}} = \operatorname{avg}(q_{17},q_{18},q_{19})$$
$$G_{\text{fragile}} = \operatorname{avg}(q_{10},1-r)$$

P_adopt and the direct AI support signal are now part of the live scoring path, but they still act as secondary modifiers rather than replacing task evidence.

Cluster-Level Transformation Scoring

For each task cluster i, the engine loads a base exposure prior $e_i^{(0)}$, a base augmentation prior $a_i^{(0)}$, and a base automation prior $u_i^{(0)}$. The automation prior is the mean of the partial-automation and high-automation priors. The current implementation also blends the cluster exposure prior lightly with the occupation-level exposure prior $E_{\text{occ}}$.

Let $h_i$ be the built-in human-advantage weight for cluster i, let $k_i = \mathbf{1}[i \in \text{roleCriticalClusters}]$, and let $T$ be the direct AI/tool support signal. The live formulas are:

Exposure, augmentation, automation
$$\bar{e}_i^{(0)} = \operatorname{avg}(e_i^{(0)}, E_{\text{occ}})$$
$$R_{\text{adopt}} = 0.92 + 0.16P_{\text{adopt}}$$
$$e_i = \operatorname{clamp}\!\left(\bar{e}_i^{(0)} \cdot (0.68 + 0.62E)\cdot R_{\text{adopt}}\cdot(1 + 0.08k_i)\cdot(1 - 0.42 h_i C),\,0,\,1\right)$$
$$a_i = \operatorname{clamp}\!\left(a_i^{(0)} \cdot (0.72 + 0.48A_{\text{aug}} + 0.32T)\cdot(1 + h_i(0.16 + 0.12T))\cdot R_{\text{adopt}},\,0,\,1\right)$$
$$u_i = \operatorname{clamp}\!\left(u_i^{(0)} \cdot (0.68 + 0.68A_{\text{auto}} + 0.18P_{\text{adopt}})\cdot(1 - h_i(0.42C + 0.18T)),\,0,\,1\right)$$
$$m_i = \arg\max\{a_i,\ u_i\}$$

The model then estimates absorbed, retained, and elevated share for each cluster.

Bundle transformation
$$\text{absorbed}_i = s_i \cdot e_i \cdot u_i \cdot (0.74 + 0.16P_{\text{adopt}})$$
$$\text{retained}_i = \max(0,\ s_i - \text{absorbed}_i)$$
$$\text{elev}_i = \begin{cases} \text{absorbed}_i \cdot (0.22 + 0.18s + 0.12r + 0.08k_i), & i \in \mathcal{E} \\ 0, & i \notin \mathcal{E} \end{cases}$$
$$t_i = \text{retained}_i + \text{elev}_i$$

Here $\mathcal{E}$ is the fixed set of elevation-capable clusters: coordination, QA/review, decision support, oversight/strategy, relationship management, and client interaction. A user-selected role-defining cluster receives extra weight in exposure and elevation so the model is not driven by time share alone.

Role-Level Aggregation

Cluster mass is aggregated into exposed share, automation/augmentation balance, strategic retained share, residual viability, and personalization fit.

Exposure and balance
$$S_{\text{exposed}} = \operatorname{clamp}\!\left(\sum_i s_i e_i,\ 0,\ 1\right)$$
$$M_{\text{auto}} = \sum_i s_i e_i u_i,\qquad M_{\text{aug}} = \sum_i s_i e_i a_i$$
$$\pi_{\text{auto}} = \begin{cases} \frac{M_{\text{auto}}}{M_{\text{auto}} + M_{\text{aug}}}, & M_{\text{auto}} + M_{\text{aug}} > 0 \\ 0.5, & \text{otherwise} \end{cases},\qquad \pi_{\text{aug}} = 1-\pi_{\text{auto}}$$
Strategic retained share
$$S_{\text{strategic}} = \sum_{i \in \mathcal{E} \cap \mathcal{R}} t_i$$
$$S_{\text{critical,exposed}} = \sum_{i \in \mathcal{K}} s_i e_i,\qquad S_{\text{critical,retained}} = \sum_{i \in \mathcal{K}} t_i$$
$$B_{\text{critical}} = \begin{cases} \operatorname{avg}\!\left(S_{\text{critical,retained}},\ 1 - \operatorname{clamp}(S_{\text{critical,exposed}}\cdot\max(\pi_{\text{auto}},0.45),0,1)\right), & |\mathcal{K}| > 0 \\ r, & |\mathcal{K}| = 0 \end{cases}$$
Residual role strength
$$V = \operatorname{clamp}\!\left(0.68 \cdot \operatorname{avg}(1-G_{\text{fragile}},\ r,\ C,\ S_{\text{strategic}},\ B_{\text{critical}},\ 1-\pi_{\text{auto}}) + 0.32 \cdot \operatorname{clamp}\!\left(\sum_i t_i,\ 0,\ 1\right),\ 0,\ 1\right)$$
Personalization fit
$$P = \operatorname{clamp}\!\left(\operatorname{avg}(A_{\text{occ}},\ S_{\text{strategic}},\ S_{\text{critical,retained}} \text{ or } r,\ C,\ F_{\text{adapt}},\ T,\ 0.55s + 0.45r,\ 1-G_{\text{fragile}}),\ 0,\ 1\right)$$

$A_{\text{occ}}$ is the occupation-level adaptive-capacity prior from the adaptation-prior layer when available, otherwise from the occupation-prior layer.

Discrete Output Rules

The live engine uses rule-based thresholds for the five main outputs.

Role outlook

Decision tree
$$\text{Role Outlook} = \begin{cases} \text{High displacement risk}, & V < 0.34 \land \pi_{\text{auto}} \ge 0.58 \land S_{\text{exposed}} \ge 0.58 \\ \text{Role fragments}, & V < 0.42 \land S_{\text{exposed}} \ge 0.55 \\ \text{Role becomes more senior}, & S_{\text{strategic}} \ge 0.34 \land \pi_{\text{auto}} < 0.48 \land s \ge 0.50 \\ \text{Mostly augmented}, & V \ge 0.62 \land \pi_{\text{aug}} \ge 0.55 \\ \text{Routine work compressed}, & V \ge 0.48 \\ \text{Role becomes narrower}, & \text{otherwise} \end{cases}$$

Top exposed work

Selection rule
$$i^\star = \arg\max_i\left((s_i e_i)\cdot(1 + 0.35k_i)\right)$$
$$\text{Exposure tier}(i^\star) = \begin{cases} \text{high}, & e_{i^\star} \ge 0.68 \\ \text{moderate}, & e_{i^\star} \ge 0.40 \\ \text{low}, & \text{otherwise} \end{cases}$$

Mode of change

Balance thresholds
$$\text{Mode} = \begin{cases} \text{mostly automation}, & \pi_{\text{auto}} \ge 0.60 \\ \text{mostly augmentation}, & \pi_{\text{auto}} \le 0.42 \\ \text{mixed}, & \text{otherwise} \end{cases}$$

Residual role strength and personalization fit

Tiering rule
$$\text{Tier}(z) = \begin{cases} \text{strong}, & z \ge 0.68 \\ \text{moderate}, & z \ge 0.45 \\ \text{weak}, & \text{otherwise} \end{cases}$$

Confidence and Evidence Summary

The model emits multiple confidence fields instead of one monolithic uncertainty score.

Confidence fields
$$C_{\text{task}} = \operatorname{avg}_i(\text{clusterEvidenceConfidence}_i)$$
$$C_{\text{anchor}} = \operatorname{avg}(\text{occupationPriorConfidence},\ \text{selectorWeight})$$
$$C_{\text{personal}} = \operatorname{avg}(r,\ 1-G_{\text{fragile}},\ C,\ T,\ C_{\text{task}})$$
$$C_{\text{labor}} = \text{laborMarketConfidence}$$

These values are used in the evidence panel and notes. They summarize coverage and alignment; they are not posterior probabilities over the labels.

Labor-Market Context

BLS data is loaded alongside the transformation result but is not a direct driver of the main role labels. The current engine treats labor context as a separate panel.

Current design rule
$$\text{Main role labels} \perp \text{labor-market panel}$$

The context payload includes employment, annual openings, median wage, projected growth, latest mapped unemployment rate, and the last 12 monthly unemployment observations for the mapped BLS occupation group.

Current Limitations

  • Direct task inputs are still coarse. They matter more than before, but they still tilt the occupation bundle rather than fully replacing it with user-supplied time shares.
  • Adoption context is still a modifier, not a separate labor-demand model. It changes realized pressure inside the transformation engine, but it does not become an independent forecast system.
  • Occupation quality remains critical. The largest benchmark-disagreement roles still need better occupation-prior and task-bundle cleanup.
  • Confidence is descriptive. The model exposes coverage and alignment confidence, not a formal uncertainty distribution over every output.
